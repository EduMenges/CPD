A função hash escolhida é uma chave composta com p = 257. Este número foi escolhido porque é o primeiro primo maior do que 255, o maior dos valores que um byte de caractére pode assumir. ENquanto um número menor poderia ser assumido, pois a tabela ASCII cobre até o número 127, isso dificultaria o cálculo com caractéres do tipo UNICODE, que ocupam todos os valores diponibilizados em um byte. A computação é feita através do método de Horner, desta forma o risco de um overflow é minimizado.
